## 二、Qwen3-Omni-30B-A3B 测试结果

### 模型能力确认

根据模型配置文件 (`config.json`):

**✅ 支持的能力**:
- 多模态理解 (文本/图像/音频/视频)
- 音频生成 (TTS)
  - `enable_audio_output: True`
  - 支持10种语音输出语言
  - 采样率: 24000 Hz

**❌ 不支持的能力**:
- 图像生成 (无相关配置)
- 视频生成 (无相关配置)

### 服务启动

| 指标 | 数值 |
|------|------|
| 启动时间 | ~5分钟 |
| 每卡显存 | ~16.5GB |
| max_model_len | 8192 |
| Tensor Parallel | 4 |
| 状态 | ✅ 启动成功 |

### 音频生成测试

**测试方法**: 使用 vLLM Python API
**测试脚本**: `/vllm-docs/omni-test/scripts/test_audio_generation.py`

**测试状态**: ⚠️ 未完成
- 模型初始化成功
- 但音频生成部分未输出结果
- 可能原因: vLLM v0.20.2 的 Python API 可能不完全支持音频生成

## 四、关键发现

### 1. 多模态理解能力

**所有测试模型都支持**:
- ✅ 文本理解
- ✅ 图像理解
- ✅ 音频理解
- ✅ 视频理解 (需要足够的 max_model_len)

### 2. 音频生成能力

**仅 Qwen3-Omni-30B-A3B 支持**:
- ✅ 配置文件确认支持 (`enable_audio_output: True`)
- ✅ 有完整的 TTS token 配置
- ⚠️ 实际测试未完成 (API限制?)

### 3. 图像/视频生成能力

**所有模型都不支持**:
- ❌ 这是理解模型，不是生成模型
- ❌ 需要使用 Stable Diffusion、Sora 等专用模型

### 4. 性能对比

| 模型 | 卡数 | 文本速度 | 视频支持 | 音频生成 |
|------|------|----------|----------|----------|
| Qwen2.5-Omni-7B | 1卡 | 30-38 tok/s | ❌ | ❌ |
| Qwen2.5-Omni-7B | 4卡 | 29-33 tok/s | ✅ | ❌ |
| Qwen3-Omni-30B-A3B | 4卡 | ~25 tok/s (预估) | ✅ | ✅ (配置) |

## 五、使用建议

### 场景推荐

| 应用场景 | 推荐模型 | 推荐配置 |
|----------|----------|----------|
| 文本对话 | Qwen2.5-Omni-7B | 单卡, max_len=2048 |
| 图像理解 | Qwen2.5-Omni-7B | 单卡, max_len=4096 |
| 音频理解 | Qwen2.5-Omni-7B | 单卡, max_len=8192 |
| 视频理解 | Qwen2.5-Omni-7B | 4卡, max_len=16384 |
| 高质量理解 | Qwen3-Omni-30B-A3B | 4卡, max_len=8192 |
| 语音生成 | Qwen3-Omni-30B-A3B | 4卡 (需进一步测试) |

### 配置建议

**单卡配置**:
```bash
--max-model-len 8192
--tensor-parallel-size 1
--gpu-memory-utilization 0.85
```

**4卡配置**:
```bash
--max-model-len 16384
--tensor-parallel-size 4
--gpu-memory-utilization 0.90
```
