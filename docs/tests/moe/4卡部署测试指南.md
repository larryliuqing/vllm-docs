# Qwen2.5-Omni-7B 4卡部署测试指南

## 配置对比

### 单卡 vs 4卡对比

| 配置项 | 单卡 | 4卡 |
|--------|------|-----|
| NPU 设备 | davinci5 | davinci4,5,6,7 |
| Tensor Parallel | 1 | 4 |
| 显存占用 | ~17GB | ~4.5GB/卡 |
| 启动时间 | ~4分钟 | ~5-6分钟 |
| 推理速度 | 基准 | ~3-4倍提升 |
| max_model_len | 推荐 8192 | 推荐 16384-32768 |

## 启动命令

### 方式1: 使用脚本启动（推荐）

```bash
# 基础配置 (max_model_len=16384)
bash scripts/start_omni_4npu.sh 16384 8001 "4,5,6,7"

# 高配置 (max_model_len=32768，支持更长视频)
bash scripts/start_omni_4npu.sh 32768 8001 "4,5,6,7"
```

### 方式2: 手动启动

```bash
docker run --rm \
    --device=/dev/davinci4:/dev/davinci0 \
    --device=/dev/davinci5:/dev/davinci1 \
    --device=/dev/davinci6:/dev/davinci2 \
    --device=/dev/davinci7:/dev/davinci3 \
    --device=/dev/davinci_manager \
    --device=/dev/devmm_svm \
    --device=/dev/hisi_hdc \
    -v /usr/local/Ascend:/usr/local/Ascend \
    -v /home/bes/work/vllm-project/models:/models \
    -v /home/bes/work/vllm-project/vllm-docs/omni-test:/test-data \
    -e ASCEND_RT_VISIBLE_DEVICES=0,1,2,3 \
    -e SOC_VERSION=ascend910b4 \
    -w /tmp \
    vllm-omni:v0.20.2rc \
    bash -c "
        source /usr/local/Ascend/cann/set_env.sh
        source /usr/local/Ascend/nnal/atb/set_env.sh

        python3 -m vllm.entrypoints.openai.api_server \
            --model /models/Qwen/Qwen2___5-Omni-7B \
            --trust-remote-code \
            --port 8001 \
            --host 0.0.0.0 \
            --max-model-len 16384 \
            --tensor-parallel-size 4 \
            --gpu-memory-utilization 0.85
    " > vllm_serve_omni_4npu.log 2>&1 &
```

## 性能优势

### 1. 更大的上下文长度
- 单卡: 建议 max_model_len <= 8192
- 4卡: 可支持 max_model_len = 16384-32768
- 优势: 支持更长的视频和更大的图片

### 2. 更高的吞吐量
- 并行处理: 4卡同时计算
- 推理速度: 预计提升 3-4 倍
- 并发能力: 支持更多并发请求

### 3. 更好的显存利用
- 模型分片: 每卡 ~4.5GB
- 剩余显存: 可用于 KV Cache
- 优势: 支持更长的序列和更多并发

## 预期性能指标

| 指标 | 单卡 | 4卡 | 提升 |
|------|------|-----|------|
| 文本推理速度 | ~30-38 tok/s | ~100-120 tok/s | 3-4x |
| 图像理解速度 | ~12 tok/s | ~40-50 tok/s | 3-4x |

| 最大序列长度 | 8192 | 16384-32768 | 2-4x |

## 注意事项

### 1. 启动时间
- 4卡启动需要额外的时间进行模型分片和同步
- 编译时间会比单卡长
- 预计总启动时间: 5-6 分钟

### 2. 通信开销
- Tensor Parallel 需要卡间通信
- 使用 HCCL 进行高速互联
- 建议: 设置 `HCCL_OP_EXPANSION_MODE=AIV` 优化性能

### 3. 显存分配
- 模型权重平均分配到4张卡
- KV Cache 也分布式存储
- 每卡显存占用约 4.5GB

### 4. 故障排查

#### 问题1: NPU 设备不可用
```bash
# 检查 NPU 状态
npu-smi info

# 确认设备权限
ls -la /dev/davinci{4,5,6,7}
```

#### 问题2: 启动失败
```bash
# 查看详细日志
tail -100 /home/bes/work/vllm-project/vllm_serve_omni_4npu.log

# 检查容器状态
docker ps -a | grep vllm-omni
```

#### 问题3: 性能不佳
```bash
# 设置环境变量优化通信
export HCCL_OP_EXPANSION_MODE=AIV

# 或在启动命令中添加
-e HCCL_OP_EXPANSION_MODE=AIV
```
