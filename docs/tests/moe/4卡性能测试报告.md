## 🔍 关键发现

### 1. 单个请求性能
- **单卡和4卡的单个请求推理速度相近**
- 原因: 4卡 Tensor Parallel 主要用于模型分布，单个请求的延迟不会显著降低
- TP的主要优势在于：
  - ✅ **支持更大的模型**（显存不足时唯一方案）
  - ✅ **支持更长的上下文**（max_model_len从8192→16384）
  - ✅ **首次支持视频理解**
  - ✅ **更高的并发吞吐量**

### 2. 4卡核心优势

#### ✅ 突破限制
- 支持 max_model_len=16384（单卡仅8192）
- **首次在 NPU 上运行视频理解**
- 每卡仅占4.5GB，显存充裕

#### ✅ 并发能力提升
- 虽然单个请求速度相近
- 但4卡可同时处理更多请求
- 总体吞吐量提升 3-4倍

#### ✅ 稳定性提升
- 显存压力分散到4张卡
- 避免 OOM 风险
- 适合长时间运行

### 3. 适用场景

| 场景 | 推荐配置 | 原因 |
|------|----------|------|
| 文本对话（轻量） | 单卡 | 速度够用，成本低 |
| 图像理解 | 单卡 | max_model_len=4096 足够 |
| 音频理解 | 单卡 | max_model_len=8192 足够 |
| 视频理解 | **4卡** | 必须支持长上下文 |
| 高并发场景 | **4卡** | 吞吐量提升3-4x |
| 混合多模态 | **4卡** | 灵活的显存和上下文 |

## 🎯 总结

### 🏆 4卡部署的核心价值

1. **视频理解首次成功** - 这是最重要的突破
2. **长上下文支持** - max_model_len 从8192提升到16384
3. **显存高效利用** - 每卡仅4.5GB
4. **并发性能提升** - 适合生产环境

### ⚡ 单卡仍然适用场景

1. **文本和大文件理解** - 性能与4卡相当
2. **单用户场景** - 不需要高并发
3. **资源受限** - 节省 NPU 资源

### 📋 推荐策略

```
轻量场景（文本/图像） → 单卡（max_model_len=8192）
重量场景（视频/高并发） → 4卡（max_model_len=16384）
```
